# Codex SDK 技能 (Skill) 挂载与加载机制说明

## 1. 核心理念：约定优于配置

Codex 放弃了在代码中通过显式 API（如 `codex.registerSkill()`）动态注册技能的传统模式，转而采用**基于文件系统的静态发现机制**。
只要在特定的工作目录中按照规范放置技能文件夹，Codex 在启动和运行阶段就会自动接管，实现无缝挂载。

## 2. 作用域与路径识别

Codex 会根据技能文件夹所处的相对路径，自动区分技能的作用域：

* **项目级技能 (Project-Level)**
  * **路径:** `[当前工作区]/.agents/skills/`
  * **场景:** 针对特定项目的专属能力。例如，在 `lca-workspace` 项目中运行 Coda 时，该目录下的技能只会服务于当前项目。这非常适合存放随项目代码库一起通过 Git 进行版本控制的自动化脚本或测试规范。
* **全局技能 (Global)**
  * **路径:** `~/.agents/skills/` (或系统指定的主目录)
  * **场景:** 跨项目通用的个人工具链。无论在哪个目录下唤起 Codex，这些通用技能（如全局日志分析、环境诊断）都始终可用。

## 3. 标准的技能内部结构

为了让 Codex 能够成功识别并调用，每个技能必须是一个独立的文件夹，且必须包含核心的元数据文件。以一个采用 Local-first Dual-Engine 架构的知识库管理技能为例，其结构应当如下：

```text
.agents/skills/
└── wiki-manager-skill/            # 技能目录名称（通常作为技能ID）
    ├── SKILL.md                   # 核心元数据与系统指令（必须）
    ├── file-listener.js           # 自定义执行脚本：文件系统监听
    └── compile-to-sqlite.js       # 自定义执行脚本：Markdown 编译与向量入库
```

### 核心解析：`SKILL.md`

这是 Codex 识别技能的唯一入口。它必须包含 YAML 格式的 Frontmatter（用于元数据声明）和 Markdown 正文（用于具体指令）。

```markdown
---
name: wiki-manager-skill
description: 当需要检索本地知识库、查阅架构规范或更新文档时触发。此技能通过读取 SQLite 向量数据库提供高精度的上下文。
---
# Instructions
当调用此技能时，请执行同目录下的 `compile-to-sqlite.js` 来获取最新的知识库状态，并分析 Markdown 文件的编译结果...
```

## 4. 惰性加载机制 (Lazy Loading) 的完整生命周期

为了最大程度地节省大模型的 Token 消耗，并保持上下文的极简，Codex SDK 采用分步式的惰性加载策略：

### 阶段一：扫描与元数据注入（SDK 初始化 / 线程启动时）

1. 当通过 SDK 执行 `new Codex()` 并启动线程时，底层进程会立即扫描工作目录下的 `.agents/skills/`。
2. 它**不会**读取脚本的源代码或 `SKILL.md` 的正文。
3. 它仅提取每个技能的 `name` 和 `description`，并将其作为轻量级的“可用工具列表”悄悄注入到智能体的系统上下文中。

### 阶段二：意图识别与自主决策（用户输入 Prompt 时）

1. 用户通过 `thread.run()` 下达自然语言指令。
2. Codex 会将用户的需求与阶段一注入的 `description` 列表进行交叉比对。
3. 如果需求与某个技能匹配（例如，用户要求查阅系统架构规范，触发了检索知识库的意图），模型内部会决定调用该技能。

### 阶段三：全量读取与执行（触发技能时）

1. Codex 定位到目标技能文件夹，此时才会完整读取 `SKILL.md` 中的 `# Instructions`。
2. 智能体会根据指令中的要求，在沙箱环境中执行对应的本地脚本（如拉起 SQLite 数据库连接并执行检索逻辑）。
3. 脚本的标准输出 (stdout) 将被返回给模型，模型整合这些信息后生成最终的回答或执行后续的代码操作。

## 5. 机制优势总结

* **零代码集成:** 开发者无需在 SDK 层面修改任何代码即可为智能体热插拔新能力。
* **按需加载:** Token 消耗极低，即使本地挂载了数百个技能，只要未被触发，就不会污染核心对话的上下文窗口。
* **Git 友好:** 技能本身作为纯文本和脚本存在于 `.agents/` 目录中，天然支持团队协作与版本回溯。
